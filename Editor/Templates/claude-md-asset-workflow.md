## Asset Workflow

This section is managed by the **Claude Unity Bootstrap** package. Edit anything *outside* the sentinel comments freely; content *inside* the comments is regenerated by `Window > Claude > Bootstrap` whenever the template revision changes.

Two channels, picked by what the asset *is*:

| Channel | What it covers | Who acts |
|---|---|---|
| **Unity MCP** ([CoplayDev/unity-mcp](https://github.com/CoplayDev/unity-mcp)) | Anything serializable as Unity data: scenes, prefabs, ScriptableObject instances, materials, GameObjects, components, lighting, NavMesh, scene hierarchy, asset moves/deletes. | Claude calls MCP tools directly (`manage_scene`, `manage_prefabs`, `manage_scriptable_object`, `manage_material`, `manage_components`, `manage_gameobject`, etc.). No pause, no prompt file. |
| **Unity AI Assistant prompts** | Genuinely generative art: rigged models, painterly textures, VFX flipbooks, audio, music, UI illustrations. | Claude writes a flat numbered prompt file under `Assets/_Prompts/`, pauses, and the user runs it through the **Window > Claude > Prompt Runner** window, which sends it to Unity AI Assistant in Agent mode. |

### MCP-first rule

**Default to Unity MCP for any in-engine asset work.** If `mcp__UnityMCP__*` tools are loaded in the current Claude Code session, use them — do **not** write a prompt file for something MCP can do. A prompt file is only the right answer when the asset is generative art Unity AI Assistant must produce.

If MCP isn't currently loaded (`mcp__UnityMCP__*` tools absent from the deferred-tools list), open `Window > Claude > Bootstrap` and walk the checklist — the wizard installs CoplayDev's bridge, writes `.mcp.json`, and starts the server.

### What goes via Unity MCP tool calls

Everything serializable. Claude calls the MCP server directly — no pause, no prompt file:

- **Scenes** — composition, hierarchy organizers, NavMesh bake, lighting, skybox / ambient settings, Volume profiles.
- **Prefabs** — players, creatures, UI, environment, VFX containers, including component wiring (which scripts attach, which serialized fields point where, event-channel references).
- **GameObjects in a scene** — lights, ground / terrain, props, cameras, audio sources, hierarchy moves.
- **ScriptableObject `.asset` instances** — every kind of data asset; includes setting all serialized field values.
- **Materials** — URP `.mat` files, color / smoothness / metallic, shader assignment.
- **Asset moves, renames, deletes** — keeps `.meta` files in sync.

### What requires Unity AI prompts (flat, numbered, deleted after run)

Only generative art that the MCP server cannot produce:

- **Rigged models** — characters, creatures, vehicles, props.
- **Stylized environment art** — trees, rocks, ruins, water decorations.
- **Texture sets** — bark, rocks, foliage atlases, anything painterly.
- **VFX flipbooks** — effects, particle textures, animated UV sheets.
- **Audio** — ambient loops, vocalizations, combat SFX, magic SFX.
- **Music** — area themes, combat tracks, stings.
- **UI illustrations** — portraits, ability icons, confirmation art.

### Prompt convention

Prompts live **flat** under `Assets/_Prompts/`, with no category subfolders. Filenames are `NN-name.md` where `NN` is a zero-padded execution order (`01-`, `02-`, …). Each prompt is temporary: once the user has run it through Unity AI Assistant, the **Prompt Runner** window deletes the file (and its `.meta` sibling) — `_Prompts/` should only ever contain pending work.

Each file follows this skeleton:

~~~markdown
# <asset name>

**Type:** <model | texture | vfx | audio | music | ui-illustration>
**Intended use:** <where the asset will be used>
**Style anchors:** <art direction summary>

## Prompt

<the prompt body to send to Unity AI Assistant>

## Acceptance criteria

- <what the result must look like>
- <integration notes — rig type, texel density, loop length, etc.>
~~~

When writing a new batch, renumber to keep ordering correct. Leading zeros so `ls` sorts properly. Dependencies between prompts go in the body, not just in the filename order.

### Running prompts

The **`Window > Claude > Prompt Runner`** window scans the prompts folder (default `Assets/_Prompts/`, configurable via EditorPrefs) and exposes:

- **▶ Run Next** — sends the selected prompt's content to Unity AI Assistant via the public `AssistantApi.Run(...)` API in Agent mode. The Assistant window opens with the prompt submitted; approve any tool permission dialogs.
- **✓ Mark Done & Delete** — removes the `.md` (and `.meta`) and advances to the next prompt.
- **↓ Skip** — move to the next prompt without deleting (use when an earlier asset must land first).
- **Refresh** — re-scan the folder.

Per-prompt user effort is **two clicks**: Run Next → wait for Assistant → Mark Done & Delete.
